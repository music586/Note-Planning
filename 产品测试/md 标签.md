# **微记：一款极简笔记软件的诞生**

一款好的笔记软件，不应该让人花时间研究软件本身，而应该帮助人更快地记录、理解和行动。

这是一个关于极简笔记软件 **“微记”** 的产品开发记录。

它既是一篇可以正常阅读的文章，也是一份 Markdown 综合语法测试材料。文档涵盖标题、段落、列表、引用、链接、图片、代码、表格、任务列表、脚注、数学公式、流程图和 HTML 等语法。

## **1. 为什么要做一款新的笔记软件**

现在的笔记软件越来越强大。

有的软件提供数据库，有的软件支持双向链接，还有的软件集成了白板、日历、任务管理和人工智能。功能不断增加，但用户的注意力却越来越分散。

我们提出了一个问题：

当一个人只想快速记下一件事时，软件是否真的需要那么复杂？

“微记”的目标不是成为一个无所不能的知识管理系统，而是解决三个简单问题：

- 快速记录。

- 清晰整理。

- 随时找到。

这三个目标看起来普通，却决定了产品的大部分设计。

## **1.1 产品原则**

产品团队最终确定了以下原则：

- **先记录，后整理**

- **内容优先，界面退后**

- **默认简单，能力渐进**

- **本地优先，数据可控**

- **格式开放，长期可迁移**

其中最重要的一条是：

用户应该拥有自己的内容，而不是被某一种软件格式绑定。

因此，软件将 Markdown 作为主要存储格式。

### **1.1.1 为什么选择 Markdown**

Markdown 有几个明显优势：

- 文件本质上是纯文本。

- 可以使用任何文本编辑器打开。

- 适合 Git 管理和版本比较。

- 容易被人工智能和程序解析。

- 即使软件停止维护，内容仍然可以继续使用。

Markdown 文件通常使用 `.md` 或 `.markdown` 作为扩展名。

## **2. 一次真实的记录过程**

星期一早晨，产品经理打开软件，写下当天最重要的三件事：

- 检查启动速度

- 完成搜索功能原型

- 与设计师确认深色模式

其中第一项已经完成，另外两项仍待处理。

任务还可以包含层级：

- 完成搜索功能

- 设计搜索输入框

- 支持标题搜索

- 支持正文搜索

- 增加搜索结果高亮

在产品设计中，任务状态不应该只有“完成”和“未完成”。

有时还需要表示：

- `TODO`：尚未开始

- `DOING`：正在进行

- `BLOCKED`：被其他问题阻塞

- `DONE`：已经完成

例如：

```text
TODO     设计标签系统
DOING    优化编辑器性能
BLOCKED  等待同步接口
DONE     完成应用图标初稿
```

## **3. 文本格式测试**

Markdown 可以为文字增加不同层级的强调。

普通文字保持自然阅读状态。

*这是一段斜体文字。*

*这也是一段斜体文字。*

**这是一段粗体文字。**

**这也是一段粗体文字。**

***这是一段粗斜体文字。***

***这也是一段粗斜体文字。***

这是一段删除线文字。

***这是一段同时包含粗体、斜体和删除线的文字。***

在产品文档中，可以使用粗体突出**关键结论**，使用斜体表示*补充说明*，使用删除线标记已经废弃的方案。

某些 Markdown 编辑器还支持以下扩展：

这是一段高亮文字。

H2O 表示水。

2^10^ 等于 1024。

这些语法不是 CommonMark 标准，不同编辑器的渲染结果可能不同。

也可以使用 HTML 实现类似效果：

HTML 高亮文字

水的化学式是 H2O。

平方公式中会出现 x2。

## **5. 列表系统**

### **5.1 无序列表**

无序列表可以使用短横线：

- 产品设计

- 技术开发

- 用户测试

也可以使用星号：

- 编辑器

- 搜索

- 数据同步

还可以使用加号：

- 浅色模式

- 深色模式

- 跟随系统

为了保持文档一致，建议在同一项目中统一使用一种符号。

### **5.2 有序列表**

一个正常的产品开发流程可能包括：

- 明确问题

- 定义目标

- 设计方案

- 开发实现

- 测试验证

- 发布迭代

Markdown 源文件中的数字不一定要连续：

- 第一步：收集需求

- 第二步：分析需求

- 第三步：验证需求

渲染器通常会自动生成连续编号。

也可以从指定数字开始：

- 第五阶段：小范围测试

- 第六阶段：公开发布

- 第七阶段：持续优化
